Unit 4: Functions in C

1. Introduction to Functions

A function is a block of code that performs a specific task. Functions make code modular, reusable, and easier to debug.

2. Advantages of Using Functions

  • Reusability of code
  • Improves readability and structure
  • Makes testing and debugging easier
  • Breaks down large problems into smaller ones

3. Types of Functions

  • Library Functions: Predefined functions provided by C (e.g., printf(), scanf(), sqrt())
  • User-defined Functions: Functions written by the programmer

4. Syntax of a Function

return_type function_name(parameter_list) {
    // body of the function
}

Example: A function to add two numbers might look like:

int add(int a, int b) {
    return a + b;
}

5. Function Declaration (Prototype)

The declaration tells the compiler about the function name, return type, and parameters before its actual definition.

int add(int, int);

6. Function Definition

The function definition contains the actual code to perform the task. This is written once, typically after the main() function or before it.

7. Function Call

To use a function, we call it from main() or another function using its name and passing required arguments.

int result = add(5, 10);

8. Categories of Functions (Based on Arguments and Return Type)

CategoryDescription
1. No arguments, no return value Does not take inputs or return a value
2. Arguments but no return value Takes input but does not return a value
3. No arguments but return value Returns a result without input
4. Arguments and return value Accepts input and returns output

9. Scope of Variables

  • Local Variables: Declared inside a function and accessible only within it.
  • Global Variables: Declared outside all functions and accessible by all.

10. Recursion

A function that calls itself is called a recursive function. It must have a base condition to terminate the recursion.

int factorial(int n) {
    if (n == 0)
        return 1;
    else
        return n * factorial(n - 1);
}
